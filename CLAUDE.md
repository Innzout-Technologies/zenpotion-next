## 1. Project Overview

### Purpose
ZenPotion is a **pre-launch landing page** for a natural health beverage brand targeting the Indian market. The primary goal is waitlist lead capture — users enter name/email/phone before the product ships. A secondary goal is brand storytelling through ingredients and benefits content.

### Key Features
- Animated hero with floating bottle illustration (CSS + Framer Motion)
- Parallax scroll section with floating ingredient emojis
- Ingredients showcase (Coconut, Lemon, Mint, Tulsi)
- Benefits grid (dark-themed cards)
- Social proof section (testimonials + waitlist stats)
- Waitlist capture form → PostgreSQL via Prisma
- Juice Runner game (interactive engagement → product recommendation)

### Current Architecture

```
Next.js 14 (App Router) + TypeScript
Tailwind CSS (utility-first + custom tokens)
Framer Motion (scroll / entrance animations)
Prisma ORM → PostgreSQL (lead/product/order schema)
Zero state management library — local useState only
Single-page layout (no dynamic routes)
```

## 3. Problems / Gaps

### Performance
- **GSAP imported but unused** — `gsap` (37 KB gzipped) is in `package.json` and likely tree-shaken, but should be removed if not used.
- **All components are `'use client'`** — No RSC (React Server Component) usage. Static sections like `Benefits`, `Footer`, `Ingredients` could be server components, reducing client JS bundle.
- **Framer Motion on every component** — each component imports `motion`, `useAnimation`, `Variants` individually. Consider a shared animation preset file to reduce duplication.
- **No image optimization in use** — `bottle.png` referenced but missing from `public/`. The bottle is currently pure CSS, which is actually great for performance.
- **No font preloading strategy** — Google Fonts loaded at runtime. Should use `next/font` properly.

### UX Issues
- **No loading skeleton** — CTA form has no loading state visual beyond a spinner.
- **Scroll-to-section nav** — Works, but Navbar links use `document.getElementById` directly — fragile, should use IDs consistently.
- **Mobile parallax** — `ParallaxSection` uses `useScroll` which can be janky on iOS Safari with momentum scrolling.
- **No error boundaries** — If Prisma/DB is unavailable, the API returns a 500 with no graceful frontend fallback.

### Scalability Concerns
- **No state management** — Fine for current scale, but as eCommerce Phase 2 rolls out (cart, auth, products), local `useState` will fragment state across components.
- **No data-fetching layer** — All API calls are one-off `fetch()` calls. No caching, retry logic, or type-safe client.
- **Single-page layout** — Products, order history, checkout will require proper routing.
- **Schema ready, UI not** — Prisma has `Product`, `Order`, `OrderItem` models but no frontend UI for them.

## 4. Improvement Suggestions

### Clean Architecture

```
app/
├── (marketing)/          # Marketing pages (RSC where possible)
│   └── page.tsx
├── (shop)/               # Future eCommerce routes
│   ├── products/
│   └── checkout/
├── components/
│   ├── ui/               # Generic, reusable (Button, Card, Badge)
│   ├── sections/         # Page sections (Hero, Benefits, etc.)
│   └── game/             # Game-specific components
├── hooks/                # Shared hooks
├── utils/                # Pure utility functions
├── lib/                  # DB, auth, email clients
└── services/             # Business logic (leads, orders, products)
```

### Reusability Improvements
1. **Extract a `<Button>` component** — the same pill-style button pattern is duplicated in Hero, CTA, Navbar. One `<Button variant="primary|outline">` component would DRY this up.
2. **Extract animation variants** — `fadeUp`, `staggerContainer` are defined in multiple components. Move to `utils/motionVariants.ts`.
3. **Extract `<SectionBadge>`** — the pill badge with dot (`Early Access Open`) pattern is reused. Make it a component.
4. **Font family inline strings** — every component repeats `style={{ fontFamily: "'Playfair Display'..." }}`. Use Tailwind's `font-display` and `font-body` classes (already configured in `tailwind.config.ts`).

### Component Splitting
- `CTA.tsx` mixes form logic + UI. Extract `useLeadForm()` hook.
- `Hero.tsx` could split the bottle illustration into `<BottleHero />`.
- `SocialProof.tsx` testimonial data is hardcoded — extract to `data/testimonials.ts`.

## 5. Game Integration — Juice Runner

### Architecture

The game is a **canvas-based endless runner** with zero external game engine dependencies.

```
State Machine (in GameSection.tsx):
  'idle' → intro screen with Start button
  'playing' → GameCanvas mounted and running
  'finished' → ResultOverlay with product recommendation

Rendering: HTML5 Canvas (60fps RAF loop)
Input: Keyboard (Space/↑) + Touch (tap to jump)
Physics: Manual gravity + AABB collision
Entities: Player, Obstacles[], Ingredients[]
```

### File Map

| File | Role |
|------|------|
| `app/components/GameSection.tsx` | State controller, section wrapper |
| `app/components/game/GameCanvas.tsx` | Canvas renderer, game loop orchestrator |
| `app/components/game/GameHUD.tsx` | Live overlay: score, lives, juice meter |
| `app/components/game/ResultOverlay.tsx` | Post-game product card + CTAs |
| `app/components/game/Player.ts` | `PlayerState` type + `createPlayer()` |
| `app/components/game/Physics.ts` | Constants + `applyPhysics()` |
| `app/components/game/Obstacles.ts` | `Obstacle` type, spawn, move |
| `app/components/game/Ingredients.ts` | `Ingredient` type, spawn, move |
| `app/hooks/useGameLoop.ts` | RAF loop, stable across re-renders |
| `app/hooks/useCollision.ts` | AABB check + hitbox shrink |
| `app/utils/gameHelpers.ts` | Product mapping + score helpers |

### Game Flow
1. Player sees idle intro → clicks "Start Game"
2. GameCanvas mounts, RAF loop begins
3. Player jumps to collect 🍊🌱💚, avoid 🍬🍕🥤
4. **Win**: collect 15 ingredients → juice meter fills → `onGameEnd(won: true)`
5. **Lose**: 3 obstacle hits → `onGameEnd(won: false)`
6. ResultOverlay maps dominant ingredient → product recommendation + "Order Now" CTA

### Integration in page.tsx
```tsx
// Inserted between Benefits and SocialProof
<GameSection />
```

## 7. Decisions & Conventions

- **`'use client'`** on all interactive components (scroll listeners, form state, canvas)
- **Inline font family strings** used throughout (Tailwind font utilities not fully applied yet)
- **Tailwind-first** — no CSS modules, no styled-components
- **Framer Motion** for all entrance/scroll animations
- **Refs over state** for game loop internals (prevents 60fps re-render cascade)
- **No class components** anywhere
- **CSS variables** in `globals.css` mirror Tailwind tokens for non-Tailwind usage contexts
